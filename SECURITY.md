# Seguranca

## Versoes suportadas

Blindfold ainda esta em fase MVP. A branch principal e a unica versao suportada.

## Reportar vulnerabilidades

Nao publiques chaves de API, tokens, logs privados ou detalhes exploraveis numa issue publica.

Se encontrares uma vulnerabilidade:

1. Usa GitHub private vulnerability reporting se estiver ativo.
2. Se nao estiver ativo, abre uma issue minima a pedir contacto de seguranca, sem detalhes tecnicos sensiveis.
3. Inclui os detalhes apenas num canal privado combinado com os maintainers.

## Boas praticas para contribuidores

- Nunca faz commit de `.env`.
- Usa `.env.example` para documentar variaveis.
- Remove chaves de API de logs, screenshots e exemplos.
- Mantem entradas do utilizador tratadas como nao confiaveis.
